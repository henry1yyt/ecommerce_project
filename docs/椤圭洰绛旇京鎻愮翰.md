# 项目答辩提纲

## 1. 选题背景

电商大促通常能提升 GMV，但也可能带来利润下降、营销投入失效、低质量用户增长和退货率上升。因此项目从经营、用户和风险三个层面展开。

## 2. 数据来源

课程 demo 共 100 条订单。为了支持用户聚类和分类模型，使用 AI 辅助设计业务规则，再通过 Python 扩展为约 6000 条模拟订单，并注入脏数据展示清洗能力。

## 3. 技术路线

数据扩展 → 清洗审计 → 派生指标 → 多维经营分析 → A/B 测试 → 同期群 → RFM + KMeans → 逻辑回归/随机森林 → HTML 报告。

## 4. 可重点展示的图表

1. 月度 GMV 趋势；
2. 品类 GMV 与退货率；
3. 渠道 ROI；
4. 同期群留存热力图；
5. KMeans K 值选择；
6. RFM 用户分群；
7. ROC 曲线；
8. 测试集风险分层退货率。

## 5. 模型亮点

- 严格区分训练、验证和测试数据；
- 阈值在验证集选择，而不是直接使用 0.5；
- 风险分层在独立测试集验证；
- 只使用下单时已知字段，避免数据泄漏；
- 输出高风险订单清单和特征重要性，可直接对应业务动作。

## 6. 业务建议

- 大促考核应从 GMV 扩展到净利润、退货率和 ROI；
- 高退货品类优先优化详情、尺码说明和质检；
- 预算向高 ROI 渠道倾斜，压缩低效投放；
- 高价值用户重点维护，沉睡用户定向召回；
- 高风险订单发货前人工确认，中风险订单加强提醒。

## 7. 项目不足

- 使用模拟数据，不代表真实企业规律；
- 缺少用户曝光和未购买样本，A/B 测试只能比较已下单用户消费；
- 没有真实库存、客服、物流节点和文本评价；
- 生产环境还需监控数据漂移和模型衰减。
